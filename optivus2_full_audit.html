BUG 1
routine_settings_sheet.dart imports from routine_tab_v2.dart — which doesn't exist in this repo
import 'routine_tab_v2.dart' — there is no routine_tab_v2.dart in lib/routine/. The RoutineFilter enum is defined in routine_tab.dart. This will throw a compile error. Fix: change import to 'routine_tab.dart'
BUG 2
lib/tabs/routine_tab.dart is a stub — home_screen.dart will show "Routine Tab" plain text for tab index 1
home_screen.dart only wires case 0 → HomeTab(). All other tabs show a Text() placeholder. lib/routine/routine_tab.dart has the real implementation but it's never imported or used. You need to import it and wire it to case 1.
BUG 3
routine_tab.dart references customTasksProvider and isPremiumProvider — neither are defined in routine_provider.dart
The real routine_tab.dart uses ref.read(customTasksProvider) and ref.read(isPremiumProvider) — these providers don't exist in providers/routine_provider.dart. They were defined in ai_routine_panel.dart in previous sessions. Either move them to routine_provider.dart or create a separate providers/app_providers.dart.
BUG 4
routine_tab.dart imports 'routine_tab_v2.dart' show RoutineFilter — that file doesn't exist
Line: import 'routine_tab_v2.dart' show RoutineFilter; — AND defines its own RoutineFilter2 enum. This creates a double enum conflict and a missing import. The RoutineFilter enum should be defined once in liquid_ui.dart or a dedicated enums.dart and imported everywhere.
WARN 1
home_tab.dart uses its own local _GlassCard — ignores liquid_ui.dart's LiquidCard
home_tab.dart defines a private _GlassCard class with color: Colors.white.withValues(alpha:0.10). This is much more transparent than liquid_ui.dart's LiquidCard (0.70 opacity). Visually inconsistent — one screen will look faded, the rest solid. Use LiquidCard from liquid_ui.dart throughout.
WARN 2
home_tab.dart uses BackdropFilter inside a ListView — GPU performance issue on Android
_HabitPill, _RoutineRow and _streakCard all use BackdropFilter + blur inside the scroll view. liquid_ui.dart's LiquidCard.solid() was specifically created to avoid this. Replace all BackdropFilter usage inside ListView items with LiquidCard.solid().
WARN 3
routine_fab_layer.dart is never wired into the app
RoutineFabLayer exists and is complete, but the real routine_tab.dart (lib/routine/routine_tab.dart) inlines its own FAB logic instead of using RoutineFabLayer. The FAB layer widget is dead code. Decide: use RoutineFabLayer as a composable OR delete it and keep the inline FABs.
WARN 4
RoutineNotifier has no seed data — first launch shows empty timeline
RoutineNotifier() initialises with const RoutineState() — all empty lists. kDefaultFixedBlocks, kDefaultSkinPlan etc. are defined as constants but never applied. Call setFixedBlocks(kDefaultFixedBlocks) in the constructor for dev/demo mode.
WARN 5
home_screen.dart doesn't use GoRouter — back navigation will break
main.dart imports go_router but MaterialApp uses home: WelcomeScreen() instead of a GoRouter config. Navigator.push() calls in login/signup screens will conflict with any future deep links or auth redirects. Wire GoRouter now before adding more screens.
WARN 6
add_task_sheet.dart defines CustomTask model locally — not in routine_provider.dart
CustomTask is defined inside add_task_sheet.dart. routine_fab_layer.dart also imports it from there. This creates a tight coupling between a UI file and a data model. Move CustomTask to routine_provider.dart alongside the other models.
WARN 7
Colors.white.withValues(alpha:x) vs withOpacity(x) mixed across files
home_tab.dart uses the newer withValues(alpha:) API. Other files use withOpacity(). Both work, but pick one and be consistent. withOpacity() is fine and more readable for now.
